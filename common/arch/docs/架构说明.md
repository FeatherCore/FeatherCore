# FeatherCore 架构支持库

## 概述

FeatherCore 架构支持库（`feathercore-arch`）为 FeatherCore 操作系统提供底层架构相关的实现。它支持多种 ARM 和 RISC-V 架构，并支持带 MMU 和不带 MMU 两种配置。

## 支持的架构

### ARM 架构

#### M 系列（微控制器）

| 架构 | 特性标志 | 处理器核心 | MMU 支持 | 典型应用 |
|-----|---------|-----------|---------|---------|
| ARMv6-M | `armv6-m` | Cortex-M0, Cortex-M0+ | 无 MMU | 超低功耗 MCU |
| ARMv7-M | `armv7-m` | Cortex-M3, Cortex-M4 | 可选 | 通用 MCU |
| ARMv7-EM | `armv7-em` | Cortex-M7, Cortex-M4F | 可选 | 带 FPU 的高性能 MCU |
| ARMv8-M 基础版 | `armv8-m-base` | Cortex-M23 | 可选 | 安全 enclave MCU |
| ARMv8-M 主流版 | `armv8-m-main` | Cortex-M33, Cortex-M55 | 可选 | 带 TrustZone 的安全 MCU |

#### A 系列（应用处理器）

| 架构 | 特性标志 | 处理器核心 | MMU 支持 | 典型应用 |
|-----|---------|-----------|---------|---------|
| ARMv7-A | `armv7-a` | Cortex-A5, A7, A8, A15 | 带 MMU | 应用处理器 |
| ARMv8-A | `armv8-a` | Cortex-A53, A72 | 带 MMU | 64 位应用处理器 |
| ARMv9-A | `armv9-a` | 最新核心 | 带 MMU | 下一代应用处理器 |

### RISC-V 架构

| 架构 | 特性标志 | 扩展 | MMU 支持 | 典型应用 |
|-----|---------|-----|---------|---------|
| RV32IMAC | `riscv32imac` | I, M, A, C | 可选 | 嵌入式 MCU |
| RV64GC | `riscv64gc` | G (I,M,A,F,D), C | 带 MMU | 应用处理器 |

## 使用方法

### Cargo.toml 配置

#### ARMv7-M (Cortex-M3/M4) 不带 MMU：

```toml
[dependencies]
feathercore-arch = { path = "../common/arch", features = ["armv7-m", "no_mmu"] }
```

#### ARMv7-A (Cortex-A7) 带 MMU：

```toml
[dependencies]
feathercore-arch = { path = "../common/arch", features = ["armv7-a", "with_mmu"] }
```

#### ARMv8-M (Cortex-M33) 带 MMU：

```toml
[dependencies]
feathercore-arch = { path = "../common/arch", features = ["armv8-m-main", "with_mmu"] }
```

#### RISC-V RV32IMAC 不带 MMU：

```toml
[dependencies]
feathercore-arch = { path = "../common/arch", features = ["riscv32imac", "no_mmu"] }
```

#### RISC-V RV64GC 带 MMU：

```toml
[dependencies]
feathercore-arch = { path = "../common/arch", features = ["riscv64gc", "with_mmu"] }
```

## 核心 API

### 架构初始化

```rust
use feathercore_arch::init;

#[no_mangle]
pub extern "C" fn arch_init() {
    init();
}
```

### 任务上下文

```rust
use feathercore_arch::TaskContext;

// 创建新任务上下文
let mut context = TaskContext::new();

// 初始化用户栈
context.init_user_stack(stack_top, entry_point, arg);
```

### 上下文切换

```rust
use feathercore_arch::{TaskContext, switch_context};

unsafe {
    switch_context(&from_task, &to_task);
}
```

### 启动第一个任务

```rust
use feathercore_arch::{TaskContext, start_first_task};

unsafe {
    start_first_task(&first_task_context);
}
```

### 跳转到内核（引导加载程序使用）

```rust
use feathercore_arch::jump_to_kernel;

unsafe {
    jump_to_kernel(vector_table_addr);
}
```

## 实现细节

### 上下文切换

#### ARM M-Profile

对于 ARMv7-M 和 ARMv8-M，使用 PendSV 异常进行上下文切换：

1. 通过设置 NVIC_ICSR 的 PENDSVSET 位触发 PendSV
2. PendSV 处理程序保存当前上下文
3. PendSV 处理程序恢复目标上下文
4. 异常返回到目标任务

对于 ARMv6-M，使用更简单的软件中断方式。

#### ARM A-Profile

A-profile 的上下文切换更复杂：

1. 保存银行寄存器
2. 处理 MMU/TLB 刷新
3. 管理缓存一致性
4. 执行异常返回或直接跳转

#### RISC-V RV32IMAC

上下文切换方式：

1. 软件中断（MSI）
2. 定时器中断
3. 直接函数调用（协作式）

#### RISC-V RV64GC

完整的上下文切换包括：

1. 被调用保存寄存器（ra, sp, gp, tp, s0-s11）
2. FPU 寄存器（f0-f31，如果启用）
3. MMU 管理（SATP 寄存器）
4. 特权模式切换（M/S/U）

### 内存管理

#### 无 MMU 配置

- 扁平内存模型
- 直接物理寻址
- 无内存保护
- 适用于嵌入式系统

#### 带 MMU 配置

- 虚拟内存支持
- 页表管理
- 内存保护
- 进程隔离
- 适用于应用处理器

### 异常处理

#### ARM 异常处理程序

- Reset 处理程序
- NMI 处理程序
- HardFault 处理程序
- MemManage 处理程序
- BusFault 处理程序
- UsageFault 处理程序
- SVCall 处理程序
- DebugMonitor 处理程序
- PendSV 处理程序（上下文切换）
- SysTick 处理程序

#### RISC-V 陷阱处理程序

- 机器模式陷阱（mtvec）
- 监督模式陷阱（stvec）
- 中断（定时器、外部、软件）
- 异常（非法指令、加载/存储错误）

## 示例

### 最小化 ARMv7-M 应用

```rust
#![no_std]
#![no_main]

use feathercore_arch::{init, TaskContext, start_first_task};

#[no_mangle]
pub extern "C" fn main() -> ! {
    // 初始化架构
    init();
    
    // 设置第一个任务
    let mut task1 = TaskContext::new();
    task1.init_user_stack(0x20001000, task1_entry as usize, 0);
    
    // 启动第一个任务
    unsafe {
        start_first_task(&task1);
    }
}

fn task1_entry(_arg: usize) {
    // 任务实现
    loop {
        feathercore_arch::wfi!();
    }
}

#[panic_handler]
fn panic(_info: &core::panic::PanicInfo) -> ! {
    loop {}
}
```

### 最小化 RISC-V RV32IMAC 应用

```rust
#![no_std]
#![no_main]

use feathercore_arch::{init, TaskContext, start_first_task};

#[no_mangle]
pub extern "C" fn rust_main() -> ! {
    // 初始化架构
    init();
    
    // 设置异常处理程序
    feathercore_arch::setup_exception_handlers();
    
    // 设置第一个任务
    let mut task1 = TaskContext::new();
    task1.init_user_stack(0x80001000, task1_entry as usize, 0);
    
    // 启动第一个任务
    unsafe {
        start_first_task(&task1);
    }
}

fn task1_entry(_arg: usize) {
    // 任务实现
    loop {
        feathercore_arch::wfi!();
    }
}

#[panic_handler]
fn panic(_info: &core::panic::PanicInfo) -> ! {
    loop {}
}
```

## 平台集成

架构库设计为与特定平台的 crate 配合使用，平台 crate 提供：

1. **内存映射定义**
2. **外设寄存器定义**
3. **中断号**
4. **向量表放置**
5. **链接脚本**

平台 crate 结构示例：

```
platform/
├── stm32f4/       # STM32F4 系列 (ARMv7-M)
├── stm32h7/       # STM32H7 系列 (ARMv7-EM)
├── raspberrypi/   # 树莓派 (ARMv7-A/ARMv8-A)
├── gd32vf103/     # 兆易创新 GD32V (RV32IMAC)
└── sifive/        # SiFive U74 (RV64GC)
```
