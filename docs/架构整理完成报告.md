## 关键设计决策

### 1. 链接脚本模板位置

**正确位置**：
- `common/arch/arm/link.x.in` - ARM 架构模板
- `common/arch/riscv/link.x.in` - RISC-V 架构模板

**模板变量**：
```
@FLASH_ORIGIN@   - Flash 起始地址
@FLASH_LENGTH@   - Flash 大小
@RAM_ORIGIN@     - RAM 起始地址
@RAM_LENGTH@     - RAM 大小
@STACK_SIZE@     - 栈大小
@HEAP_SIZE@      - 堆大小
```

### 2. 设备树解析模块

**正确位置**：`common/generated/devicetree/`

**功能**：
- 解析 `.dts` 设备树源文件
- 提取硬件配置信息
- 生成 Rust 代码到 `common/generated/src/generated/devicetree.rs`

**使用方式**：
```rust
use common::generated::devicetree::{DeviceTreeParser, generate_rust_code};

let node = DeviceTreeParser::parse_file("board.dts")?;
let rust_code = generate_rust_code(&node);
```

### 3. ARM 不同版本是否需要不同模板？

**答案**：ARM 不同版本（ARMv6-M, ARMv7-M, ARMv8-M）**可以使用同一个模板**，因为：

1. **链接脚本层面**：所有 Cortex-M 处理器使用相同的内存布局和段定义
2. **自动生成时区分**：通过设备树配置区分不同芯片的内存大小
3. **参考 cortex-m-rt**：cortex-m-rt 也使用单一模板支持所有 Cortex-M

**示例**：
```rust
// STM32F429 (ARMv7-M) 设备树配置
memory {
    flash@08000000 {
        reg = <0x08000000 0x200000>;  /* 2MB */
    };
};

// STM32H7 (ARMv7-M) 设备树配置
memory {
    flash@08000000 {
        reg = <0x08000000 0x100000>;  /* 1MB */
    };
};
```

两者使用相同的 `link.x.in` 模板，但通过设备树配置不同的内存大小。

## 构建流程

### 1. 解析设备树

```rust
// build_tool/src/generate.rs
use common::generated::devicetree::{DeviceTreeParser, generate_rust_code};

let dts_path = "platform/board/stm32/stm32f429i-disc/stm32f429i-disc.dts";
let node = DeviceTreeParser::parse_file(dts_path)?;

// 生成 Rust 代码
let rust_code = generate_rust_code(&node);
std::fs::write(
    "common/generated/src/generated/devicetree.rs",
    rust_code
)?;
```

### 2. 生成链接脚本

```rust
// build_tool/src/linker.rs
let arch = "arm";  // or "riscv"
let template_path = format!("common/arch/{}/link.x.in", arch);
let mut template = std::fs::read_to_string(&template_path)?;

// 从设备树提取内存信息
template = template.replace("@FLASH_ORIGIN@", &format!("0x{:08X}", flash_base));
template = template.replace("@FLASH_LENGTH@", &format!("{}", flash_size));
template = template.replace("@RAM_ORIGIN@", &format!("0x{:08X}", ram_base));
template = template.replace("@RAM_LENGTH@", &format!("{}", ram_size));
template = template.replace("@STACK_SIZE@", &format!("{}", stack_size));
template = template.replace("@HEAP_SIZE@", &format!("{}", heap_size));

// 写入生成的链接脚本
std::fs::write("boot/link.x", template)?;
```

### 3. 编译

```bash
# build_tool 会自动执行
cargo build --manifest-path boot/Cargo.toml \
  --target thumbv7em-none-eabihf \
  --release
```

## 参考项目

### cortex-m-rt

**链接脚本模板**：`embedded/cortex-m/cortex-m-rt/link.x.in`

**关键特性**：
- 使用 `memory.x` 提供内存布局
- 定义异常向量表
- 提供栈和堆空间
- 使用 `PROVIDE` 和 `EXTERN` 管理符号

### riscv-rt

**链接脚本模板**：`embedded/riscv/riscv-rt/link.x.in`

**关键特性**：
- 类似的模板结构
- RISC-V 特定的向量表
- 使用 `REGION_ALIAS` 简化段定义

## 总结

✅ **正确的设计**：
1. `common/generated/devicetree/` - 设备树解析模块
2. `common/arch/arm/link.x.in` - ARM 链接脚本模板
3. `common/arch/riscv/link.x.in` - RISC-V 链接脚本模板
4. 所有 ARM Cortex-M 使用同一模板，通过设备树区分具体配置
5. build_tool 负责解析设备树并生成代码

✅ **参考标准**：
- cortex-m-rt (ARM Cortex-M)
- riscv-rt (RISC-V)

这个设计遵循了 Rust 嵌入式生态的最佳实践，保证了架构的清晰性和可维护性。
